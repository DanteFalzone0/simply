Getting started

Basic syntax

The syntax is pretty easy to follow.
Just write what you want in basic English.

Here's an example:

Create a variable called $foo and set the value to "bar"
Show the contents of $foo

This will create a variable called $foo and will have the value of "bar"

The code doesn't require any specific capitalization, except when annotated.

You can have multiple statements in a single line, separated by semi-collon (;) or on a new line.

How does this work?

Certain keywords are detected within the text, then they are processed and handled accordingly.
Each action has certain keywords that will be detected, and each will be explain below.

The following words are always ignored:
a an and as by called cons constant content contents fn func function global into it me method name named of proc procedure result results string takes that the then this to value values var variable which with

While not intended, it is possible to write full sentences that are completely made of ignored word, with no negative consequences.
Instead, please use comments, as they visually indicate that something is supposed to not be part of the code.

Comments

Comments are used to write annotations about the code, to help to understand the code presented.

This way, it indicates that what is in a comment isn't supposed to be read as code.

There's 2 ways to write comments:

// Single line comments
/*
 Multi-line comments
*/
Important: everything after the // in a single-line comment will be part of the comment

Data types

There are a few data types, and they are easy to distinguish:

  • Null

    This data type simply means that there is no value.
    This data type only has 1 value: null.

  • Boolean

    This is the data type expected for binary expressions.
    It only has 2 values: true and false.

  • Character

    A Character is a single letter, number or space.
    It is defined by having any character between single quotes (').
    Example: 'A'

  • String

    A string is a set of characters.
    It is defined by having any number of character between double quotes (").
    Example: "Hello, World!"

  • Number

    This represents any number between -9007199254740991 and 9007199254740991.
    It can be positive numbers, negative numbers and have decimal points.
    The numbers can be represented in hexadecimal (preceeded by 0x), binary (preceeded by 0b) and scientific notation.
    Example: 42.0, 0xFF, -69, 1e5

    There are 2 special number values: NaN and Infinity.
    Infinity represents a value that is too large or too small.
    NaN means "Not a Number", which can be the result of some mathematical operations, like dividing 123 by "abc".

  • Array

    An array is like a container for all other data types.
    It is defined by opening a square bracket ([), putting multiple values inside (separated by comma ,) and then closing with another square bracket (]).
    You can even put variables and constants into the array.
    Example: [null, true, false, 'A', "Hello, World!", 42.0, 0xFF, -69, 1e5]

  • Function

    Please check the "Functions" section for more information.

Variables and constants

Variables and constants can be defined by just writting it in plain English.

Here are a few examples:

Define the variable $foo
Create a variable called $foo
Define $foo

Local Variables

A variable is just where you store data. This can then be used in calculations and show on screen.

A local variable is defined by starting with $, followed by letters and numbers and underscore (E.g.: $foo).

Gloval variables

Global variables work the same way, but are accessible anywhere in the program.

A global variable is defined by starting with % followed by letters and numbers and underscore (E.g.: %foo).

Important: Global variables can be changed anywhere in the program.
It is a good practice to avoid them.

Constants

Similarly to global variables, constants are accessible anywhere in the program.

A constant is defined by starting with ! and end with capital letters and numbers (E.g.: !ENV).

Note: A constant value will be available all over the program.
Note: The name of a constant MUST always be in CAPITALS.
Important: Once a constant is created, you can't change the value.

Accessing Array values

To access an Array value, you can simply open and close square brackets after the variable or constant name (E.g.: !ENV["version"]).

The key between the square brackets can be a string, a character, a number, a variable or a constant.

Receiving values

To pass values into the code, you can pass an array as the second argument of the .execute() method.

All values will be available in the $argv variable.
To get the number of arguments, you can use the $argc variable.

Note: this variable is a local variable, and will be overwritten inside function.

Conditional blocks

IF Block

Keywords:
  • if
  • in case

An IF block controls the flow the code will have when executing.
If the expression that comes after results in a truthy value, the block of code after is executed.

In the absence of a scope opening, the first expression after the IF block will be executed.

Here are a few examples:

If $a then display $a
In case &can_display($foo) then display the string "Hello, World!" formatted by "%s"

Unless Block

Keywords:
  • unless

An Unless block controls the flow the code will have when executing.
It does exactly the same as the IF Block, except it executes when the expression results in a falsy value.

Here are a few examples:

Unless &fn then define &fn returns 3;
Unless $a then echo "The variable $a is missing a value."

Loops

There are several loops, to allow you to execute repeated actions.

For loop

Keywords:
  • for
  • loop from

A for loop will run a set number of times, assigning a new value to a specified variable.
In the absence of a scope opening, the first expression after the for loop will be executed.

There are a few acceptable signatures:

  • for $var from <value> to <value> [step <value>]

    The step value is optional.

  • for $var in <value>..<value>
  • loop from <value> to <value> as $var

The variable can either the a global variable or a local variable.
All values can be any statement which produces an integer.
In case the values are the result of function calls, the function will be executed ONCE before the loop.

Here are a few examples:

For $i from 0 to 10 step 2 { /* code */ }
Loop from 0 to 10 as $j { /* code */ }

Foreach loop

Keywords:
  • each
  • foreach
  • loop through

A foreach loop will go through all values in an array, and assign the key and value to a variable.
In the absence of a scope opening, the first expression after the for loop will be executed.

There are a few acceptable signatures:

  • foreach $value in <array>

  • foreach <array> as [value] $value [key $key]

    The key is optional.

  • foreach <array> as $value => $key

The variables can either the a global variable or a local variable.
All values can be any statement which produces an array.
In case the values are the result of function calls, the function will be executed ONCE before the loop.

Here are a few examples:

Foreach $number in [1, 2, 3, 4, 5] { /* code */ }
Loop through the result of calling &range(1, 10) as $number { /* code */ }

Loop control

You can control if a loop continues or stops with the following:

Continue

Keywords:
  • continue

This statement will tell the loop to go to the next iteration.

Example:

Foreach $number in [1, 2, 3, 4, 5] {
	If call &is_even($number) then continue.
	Show $number.
}
// Expected output: 1 3 5

Break

Keywords:
  • break

This statement will break out of the loop.
There won't be any new iterations after this statement is executed.

Example:

Foreach $number in [1, 2, 3, 4, 5] {
	If call &is_even($number) then break.
	Show $number.
}
// Expected output: 1

$loop variable

This variable will automatically be created for each loop.

All the values in the $loop variable
Key Description
first Will be true if it is the first iteration
last Will be true if it is the last iteration
index Contains the current iteration index
value Contains the current value for the iteration
length Contains the number of iteration that could occurr

Available functions

There are a few functions already defined. They can be overwritten, but it may break other code.
These functions are also accessible in !FN["<name>"] as well.

  • &hello_world($flags)

    • $flags - Number - 5-bit positive integer
    • Return - String

    Returns the famous "Hello, World!" text.
    The $flags argument defines how the output should be formatted based on the following bits being set:

    1. The "H" in "Hello" will be in lowercase: "hello"
    2. Removes the comma (",") after "hello"
    3. Removes the space (",") between "hello" and "world"
    4. The "W" in "World" will be in lowercase: "world"
    5. Removes the exclamation mark ("!") at the end

    Example:

    // returns "helloworld" instead of "Hello, World!"
    &hello_world(0b11111)

  • &iff($condition, $val_if_true, $val_if_false)

    • $condition - Boolean - Condition to verify
    • $val_if_true - Any
    • $val_if_false - Any
    • Return - Any

    Returns $val_if_true if $condition is true.
    Otherwise, returns $val_if_false.

    Example:

    // returns "123" if $foo is true
    &iff($foo, "123", null)

  • &is_anagram($str1, $str2)

    • $str1 - String
    • $str2 - String
    • Return - Boolean or null

    Returns true if $str1 is an anagram of $str2.
    Otherwise, returns false.
    In case $str1 or $str2 are empty, returns null.

    Example:

    &is_anagram("listen", "silent") // Returns true

  • &is_palindrome($str)

    • $str - String or Number
    • Return - Boolean or null

    Returns true if $str is a palindrome.
    Otherwise, returns false.
    In case $str is empty, returns null.

    Example:

    &is_palindrome("racecar") // Returns true

  • &is_prime($number)

    • $number - Number
    • Return - Boolean or null

    Returns true if $number is a prime number.
    Otherwise, returns false.
    In case $number isn't a number, returns null.

    Example:

    &is_prime(101) // Returns true

  • &join($array, $join_char)

    • $array - Array
    • $join_char - String or Char
    • Return - String or null

    Joins all the elements in $array, separated by $join_char, and returns the result.
    If something goes wrong, returns null.

    Example:

    // joins all received arguments, separated by a new line
    &join($argv, !EOL)

  • &len($value)

    • $value - Array or String or Char
    • Return - Number or null

    Returns the number of characters/elements in $value.
    Returns null if passed an unexpected data type.

    Example:

    // Returns the size of the EOL constant
    &len(!EOL)

  • &mirror_text($str)

    • $str - String or Char
    • Return - String

    Tries to rotate all characters that point in a direction, in $str, but rotated 180º.

    Example:

    // Returns "<hello>"
    &len(">hello<")

  • &range($start, $end, $step)

    • $start - Number
    • $end - Number
    • $step - Number - Default: 1
    • Return - Array

    Returns an array that goes from $start to $end, going by $step.
    If $start is lower than $end, OR $step is lower than 0, the range is inverted.
    Returns an empty array if $start or $end or $step (when provided) aren't Numbers.

    Example:

    // Returns [5, 3, 1]
    &range(1, 5, -2)

Opening:
  • begin
  • {

You can define a function with multiple expressions by opening a new scope.
